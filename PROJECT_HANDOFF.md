### Передаточное саммари для продолжения разработки

**Статус:** Фаза 0 — валидация. Рабочий прототип веб-диагностики собран.
**Дата:** Июль 2026
**Команда:** Vadim — продукт, разработка, Growth · Партнёр — дизайн-экспертиза, рубрика, визуальные эталоны

---

## 1. Что это (в одном абзаце)

Веб-инструмент, где инди-разработчик или соло-фаундер описывает свой бренд и оффер в свободной форме, а AI оценивает зрелость бренда по фиксированной рубрике, ставит на «лестницу уровней» и показывает конкретные разрывы. Дальше — персональный путь обучения и переоценка во времени (рост уровня = петля удержания).

---

## 2. Как мы сюда пришли (важно, чтобы не откатиться назад)

Проект начинался как **концепт ленты саммари** (TikTok-подобный бесконечный фид + 3 уровня компрессии + геймификация Duolingo). Этот концепт был **переосмыслен по трём причинам**:

1. **Ниша ленты саммари занята.** Deepstash делает ровно это (и получает жалобы на поверхностность). Headway — украинский, ~30M юзеров, тот же луп «однострочник → глубже» + стрики. Заходить в лоб = смерть.
2. **Дифференциация ушла в диагностику.** Продукт, который **рассуждает про конкретный бренд пользователя**, нельзя скопировать статичным каталогом. Это ров, который не зависит от гонки за свежестью контента.
3. **Диагностика решает проблему дистрибуции.** Она виральна по природе («мой уровень X, а твой?») — то есть механизм ценности одновременно является органическим движком привлечения. Это закрывало единственную дыру в юнит-экономике.

**Итог: диагностика — позвоночник. Лента, компрессия и геймификация — слои доставки вокруг неё.** Не возвращаться к «ленте как продукту».

## 3. Ниша и позиционирование

**Дизайн и брендинг для инди-разработчиков, соло-фаундеров и создателей приложений/игр**, которым нужно самим сделать айдентику, но нанять дизайнера не на что.

Почему именно эта ниша:
- Экспертиза — у партнёра (профессиональный график) → снимает риск генерик/уродливого AI-контента.
- Дистрибуция и авторитет — у владельца (LinkedIn, инди/геймдев-среда, свои published-игры).
- Боль денежная: плохой store-креатив прямо режет конверсию установок → подключается Growth-экспертиза («дизайн, который конвертит», а не «дизайн красивый»).
- Поле почти пустое: дизайн живёт в длинных курсах (Skillshare, Domestika, Coursera), микро-фид-ниша по дизайну для продуктовых людей свободна.

**Честная слабость, которую надо помнить:** дизайн вечнозелёный, поэтому «контент протухает → нужен AI» здесь НЕ работает как ров. Ров переносится на **персонализацию + качество исполнения (партнёр) + дистрибуцию (владелец)**.

## 4. Ядро продукта: рубрика (блокер №1)

> Черновик v0. **Финализация — за партнёром.** От неё зависят одновременно онбординг, генерация, paywall и виральный шаринг. Не кодить прод-ассессор до финализации.

**Оси (каждая 1–5):**
1. `positioning` — Позиционирование и оффер: ясно ли, кому и зачем
2. `visual` — Визуальная идентичность: система, а не случайный визуал
3. `consistency` — Консистентность на всех точках контакта
4. `differentiation` — Отстройка от конкурентов
5. `conversion` — Работает ли бренд на установки и продажи

**Уровни:**

| Ур. | Название | Признаки |
|---|---|---|
| L1 | Хаос | Есть продукт, нет бренда. Оффер не сформулирован. |
| L2 | Ремесленник | Есть название и лого, нет системы. Оффер размыт. |
| L3 | Согласованность | Есть визуальная система и понятный оффер, нет дифференциации. |
| L4 | Позиционирование | Чёткая ЦА, tone of voice, отстройка. Бренд работает на конверсию. |
| L5 | Магнит | Узнаваемый бренд как актив привлечения. |

**Правило:** общий уровень — по **слабейшим осям**, не по среднему. Бренд не сильнее слабого звена.

## 5. Что уже сделано

**Рабочий React-прототип веб-диагностики** (`BrandDiagnostic.jsx`):
- Прогрессивный онбординг в 4 шага (выбор → оффер → свободный текст → чеклист визуала) — специально против фрикшна ввода.
- Живой вызов Claude с рубрикой v0 в системном промпте, строгий JSON на выходе.
- Экран результата: тёмный «прибор» с загорающейся ступенью лестницы, оценки по 5 осям, разрывы, следующий шаг.
- Заглушки: email-вейтлист (в state), обработка ошибок вызова.

**Ограничения прототипа (переделать перед выкладкой людям):**
- Вызов Claude идёт напрямую из артефакта — это тестовый контур. **В прод только через backend-прокси, ключ на сервере.**
- Рубрика захардкожена.
- Email никуда не сохраняется.

**Оценка до soft launch:** ~270–380 ч работы ≈ **4–5 месяцев** при ~22 ч/нед. Агрессивно ~3 мес, пессимистично ~6.

## 7. Технологический стек

| Слой | Решение | Зачем |
|---|---|---|
| Веб-прототип | React + Vite | Фаза 0 |
| Мобильный клиент (позже) | Flutter + Impeller | 60/120 FPS, порт на iOS без переписывания |
| Бэкенд | Supabase (Edge Functions + Postgres + pgvector + Storage) | ключи, гейтинг, кэш, семантический матчинг |
| AI-шлюз | OpenRouter (старт) → Portkey (масштаб) | один ключ на все модели; семантический кэш, PII-редакция, версии промптов |
| Наблюдаемость AI | Helicone или Portkey | логи запросов, латентность, стоимость |
| Аналитика | PostHog | воронка, A/B, feature-флаги |
| Ошибки | Sentry | краши |
| Подписки | RevenueCat | единый API для App Store + Google Play |
| Реклама | AppLovin MAX | медиация rewarded video |

### Три правила, которые нельзя нарушать
1. **Все AI-вызовы — через backend.** Ключ на клиенте = слитый ключ.
2. **Роутинг моделей — главный рычаг маржи.** Ассессор — качественная модель (цена вопроса ~1 цент), массовая генерация ленты — дешёвая.
3. **On-demand генерация гейтится и кэшируется.** Только за sign-in / paywall / rewarded-ad + семантический кэш.

## 8. Юнит-экономика (допущения для стресс-теста, не прогноз)

### Стоимость диагностики (1000 шт., цены июль 2026)

| Модель | С кэшем рубрики | За 1 диагностику |
|---|---|---|
| DeepSeek V4 Flash | ~$0.33 | ~$0.0003 |
| Haiku-класс | ~$4.95 | ~$0.005 |
| Sonnet-класс | ~$14.85 | ~$0.015 |
| Sonnet + сжатый вывод | ~$8.85 | ~$0.009 |

**Вывод: диагностика — не статья затрат.** Можно раздавать бесплатно как лид-магнит. Когорта инфлюенсера в 500–2000 человек = $1–30 суммарно.

Рычаги: кэшировать рубрику (она идентична в каждом вызове, скидка до 90% на кэш-ридах) + держать вывод строго структурным (выход доминирует в счёте).

### Экономика приложения

| Параметр | База | Оптимизм (ниша) |
|---|---|---|
| Free → Paid | 3% | 6% |
| Чистый вклад / 1000 установок | ~$1 000 | ~$2 400 |
| Комиссия стора | 15% (small business) | — |

**Ключевые выводы:**
- **Платный UA — ловушка.** CPI $2–5 > blended LTV/установка $1–2.4. Только органика.
- Порог «замены зарплаты»: ~1 500–2 000 органических установок/мес (в оптимизме ниши ~700–900).
- Оба рычага — конверсия и LTV. На них бьёт персонализация и узость ниши.

## 9. Дистрибуция: B2B2C через инфлюенсеров

**Решает проблему №1** (CAC), потому что приносит тёплую, предквалифицированную, уже платящую аудиторию.

**Пилот с pruddan:** ко-брендированная диагностика бесплатно покупателям его курса, как пост-курсовый компаньон («понял ли ты материал»). Ценность для него: завершаемость, результаты студентов, вау-интерактив, отстройка, сигнал кто не усвоил.

**Решение по скоупу:** захардкодить фреймворк pruddan как рубрику v1. **Не строить конфигурируемый мульти-авторский движок до первой доказанной сделки.**

**Три предостережения:**
1. Питч строить вокруг ЕГО ценности, не своей.
2. IP и данные чисто: его методология + ответы студентов, не переиспользовать для конкурентов.
3. Не дать чужой рубрике подменить своё ядро — это отдельный конфиг.

**Открытая развилка (решать позже, не сейчас):** Путь A — консьюмер-приложение, инфлюенсеры как канал. Путь B — конфигурируемый диагностик-движок для авторов курсов (B2B2C-инфраструктура). Не взаимоисключающие; B может профинансировать A.

## 10. Юридика и комплаенс

- **Копирайт:** контент — пересказ своими словами, без цитирования крупных кусков; продукт как самостоятельный аналитический материал.
- **Товарные знаки:** фокус на методологиях и паттернах, а не на защищённых названиях.
- **Приватность:** описания брендов пользователей — конфиденциальны. Privacy Policy + ToS/EULA на внешнем хостинге.
- **Финансы:** биллинг Google Play (15%, small business program); ФОП 3 группа, КВЕД на издание ПО и информационные услуги.

## 11. Риски

| Риск | Митигация |
|---|---|
| Ассессмент воспринимается как генерик → потеря доверия | жёсткая рубрика + human-in-the-loop калибровка партнёром + кнопки «точно/мимо» |
| Фрикшн ввода убивает activation | прогрессивный онбординг, ценность до paywall |
| On-demand генерация съедает маржу | гейтинг + семантический кэш |
| Зависимость от партнёра | рубрика гейтит Фазу 0, визуал гейтит Фазу 3 — это критический путь, а не «иногда поможет» |
| Flutter — не домашний стек | самый большой неизвестный в оценке сроков; веб-first это откладывает |
| Скоуп ползёт | всё из «не в MVP» остаётся вне MVP |
