# Домашка и шкала оценивания

Как это работает: после урока человек видит задание (`task` урока), пишет ответ по своему продукту и жмёт «Сдать на проверку». Модель оценивает ответ из 10 баллов **строго по фиксированным критериям** — оценка объективна и воспроизводима, потому что критерии заданы вручную, а не генерируются.

## Где править (файл `supabase/functions/api/homework.ts`)

**1. `DEFAULT_CRITERIA`** — универсальная сетка на 10 баллов, применяется ко всем урокам, у которых нет своей. Сейчас: по делу (4) + конкретика (3) + применимость (3).

**2. `HOMEWORK_OVERRIDES[axis][lessonIndex]`** — своя сетка под конкретный урок. Уже заполнены предварительные щедрые критерии по всем осям и урокам (правь под себя). Формат:

```ts
marketing: {
  0: {
    prompt: "Текст задания (необязательно, по умолчанию берётся task урока)",
    criteria: [
      { label: "Что проверяем", points: 4, hint: "что считать выполненным (подсказка модели)" },
      { label: "…", points: 3 },
      { label: "…", points: 3 },
    ],
  },
}
```

Правила: сумма `points` по критериям = 10. `hint` необязателен, но помогает модели ставить одинаковый балл за одинаковые ответы.

`axis` — это слепая зона из диагноза: `marketing`, `brand`, `product`, `operations`. `lessonIndex` — номер урока с нуля (порядок из `PLAN_BY_AXIS` в `course.ts`).

## Как считается балл

Модель ставит по каждому критерию 0..max. Сервер сам суммирует и клампит (не даёт превысить максимум) — модели доверяем только «сколько засчитать за критерий», итоговую арифметику считаем в коде.

**Тон на бете — мягкий, не строгий** (по просьбе). Проверяющий — доброжелательный наставник: за искреннюю попытку про свой продукт ставит щедро, 0 только если пусто или совсем не по теме, частичный ответ — минимум половина. Когда захочешь строже — ужесточи правила в системном промпте `gradeHomework` (сейчас там «давай презумпцию доверия, поощряй попытку»).

## Что осталось на потом

- Задания сейчас — это `task`, который генерирует модель под нишу. Если хочешь полностью статичные формулировки заданий — задавай `prompt` в override.
- Сохранение оценок в БД (пока балл только показывается ученику) — добавить, когда будут аккаунты.
