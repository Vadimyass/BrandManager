# Go-live: вывод в прод

Чек-лист перехода с тестового контура (бесплатная Gemma) на продакшн.

## 1. Переключить модели на прод

Сейчас роли переопределены секретами на бесплатную Gemma. Для прода — вернуть на нормальные модели. Сверь актуальные ID на openrouter.ai/models (иконка копирования у модели).

```bash
cd brand-diagnostic
supabase secrets set \
  MODEL_GATE=deepseek/deepseek-chat \
  MODEL_ASSESSOR=anthropic/claude-sonnet-4.6 \
  MODEL_VALIDATOR=anthropic/claude-haiku-4.5
supabase functions deploy api --use-api
```

Альтернатива — самообновляющиеся алиасы `~anthropic/claude-sonnet-latest` и `~anthropic/claude-haiku-latest` (не устаревают, но цена может меняться — следи).

Проверить, что ключ OpenRouter пополнен (не free-баланс): openrouter.ai → Credits.

## 2. Поставить лимит трат (защита от абуза)

Функция публичная (`verify_jwt = false`, CORS `*`) — это осознанно, диагностика раздаётся как лид-магнит. Риск: кто-то дёргает её массово и жжёт твои токены. Backstop — лимит на ключе:

- openrouter.ai → Keys → на ключе выставить дневной/месячный `Credit limit` (например $20/день). Упрётся — вернёт 402, но не разорит.

## 3. Проверить прод вживую

- Пройти диагностику на 2–3 реальных бизнесах разных ниш → оценки осмысленные, без жаргона.
- Открыть курс, пролистать блок → уроки читаются, JSON не ломается (на Sonnet это стабильно).
- Оставить почту двумя путями → строки появились в `waitlist` с `intent`.
- Кнопки «точно/мимо» → пишутся в `diagnostics.feedback`.
- Проверить с телефона: свайпы, кнопки вариантов, кнопка курса.

## 4. Данные и наблюдаемость

- Supabase → Table Editor: `diagnostics` (полный лог + usage + latency), `waitlist`.
- Калибровка: фильтр `feedback = 'miss'` → читать `input` против `result`.
- Готовность платить: `waitlist` где `intent = 'purchase'`.

## Что НЕ входит в этот go-live (осознанно отложено)

- Приём платежей (Lemon Squeezy/Paddle) — чекаут пока собирает почту, не берёт деньги.
- Подписка и аккаунты (Фаза 3, см. COURSE_PROGRAM.md).
- Кэш уроков по нише (уровень 2–3) — внедрять, когда объём начнёт упираться в токены.
- Картинки к урокам от партнёра.
- Финализация рубрики/книжного контента партнёром.

## Состояние на момент go-live

Прод-код готов: цепочка 4 агентов (Фаза 1) + курс 15 уроков в 3 блоках (Фаза 2), устойчивость к вкладкам и 429, кэш промптов уровня 1, честный чекаут с замером намерения. Деплой: фронт на GitHub Pages (vadimyass.github.io/BrandManager), бэкенд Supabase Edge Function.
